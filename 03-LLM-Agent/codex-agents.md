

# AGENTS.md

## 环境
- 在本仓库工作必须使用 conda 环境：`deepconf`

## 分支
- 首先判断你的工作是否需要改代码或者新增代码。如果不需要的话，忽略这一节
- 禁止在 `main`/`master` 上直接改代码或提交。
- 开工前先 `git branch` 检查；若在 `main`/`master`，先新建并切换分支。
- 分支命名：`2026-<MM>-<DD>-<feature-name>`（例：`2026-02-15-different-k`）

## 产物目录
- 本次任务产生的**所有新文件/实验输出**必须放到：
  - `2026-<MM>-<DD>-<feature-name>/`
- 默认在该目录下提供：`report.md`（必要时再加 `commands.sh`/`reproduce.md`）

## Report
- 每个任务必须写 `report.md`，内容要**可审计/可复现**：
  - 实现 feature：列出改动的文件 + 行号范围（或关键 diff）
  - 跑实验：给出完整复现命令 + 输出路径
- **写得尽量简洁**：只写结论、改动点、命令、路径；不要写废话。

你如果跑一个超过 5 分钟的实验，你在跑之前去 report.md 里边写明白这个实验要跑多久。

当这个实验跑完了之后，你再去 report.md 里边写一下这个实验实际上跑了多久
